Welcome

This tutorial introduces you to markdown, a plain-text syntax for creating formatted documents.

Learning Objectives

Upon completion of this session, you will be able to:

  • Create and view a markdown document;
  • Add section headers;
  • Add bold, italic and typewriter formatting to text;
  • Create bulleted and numbered lists;
  • Add URLs and images.

Markdown

Markdown offers a means of creating format-rich, production-quality documents using only a text editor. That is valuable because text files can be edited and viewed on nearly every computer on the planet. The same cannot be said for Word documents (that require licenced software) or PDFs (that require compilation from an input file). The power of markdown will really be revealed in later tutorials when we create documents that contain R and Stata output. The goal of this tutorial is to get you familiar with the elements that comprise the markdown format. We will build on those skills to report statistical analyses later.

CommonMark

CommonMark is standardised variant of Markdown. It is the basis of the markdown used by GitHub to describe code packages, including the bootcamp package that we created for your training.

The CommonMark website provides a cheatsheet that demonstrates the core markdown elements:

https://commonmark.org/help/

It also provides a fantastic interactive tutorial for learning markdown. No point re-inventing the wheel here. Launch their tutorial now and do not return here until you have completed it:

https://commonmark.org/help/tutorial/

Exercise

Within RStudio, create a new text file by clicking File -> New File -> Text File. Save the file as MyFile.md; it is important that you specify the file extension as md.

Now give the document some content. The content need only be limited by your own creativity, so go nuts and give us a glimpse of your personality.

Please add:

  • A header
  • a horizontal rule
  • two paragraphs of text (unless you are effortlessly verbose, you may find lorem ipsum useful here. Google it.)
  • a numbered or bulleted list
  • an image
  • a working link to some website (test the link!)

Pandoc

Pandoc is a tool that converts documents of one type into another type. It is a command line tool, meaning you interact with it using Terminal or DOS or the bash shell or PowerShell or (…yawn). The quality of the conversion, however, depends on the format you are converting from. Markdown is a good base-case of document because it converts to everything nicely.

To illustrate the usage of pandoc, imagine you have a file called Cats.md and you want a Word version called Cats.docx. You would run:

pandoc Cats.md -s -o Cats.docx

The first argument to the pandoc application is the document we are converting from. The -s option means “I want the output to be fully-fledged or stand-alone”. The -o options means “And here comes the name of file I want you to create”. By default, pandoc infers the output format by the file extension. Thus, we have not told it to create a Word file but it knows that .docx files are Word files.

Brilliant thing number 3,281 about RStudio is that it ships pandoc by default, and provides a Terminal pane for interacting with the computer running your R session. Finding pandoc, however, can require some tenacity.

RStudio-Cloud

If you using RStudio-Cloud, pandoc is (at Aug-2019) stored in

/usr/lib/rstudio-server/bin/pandoc/pandoc

but the program is not on the path (the list of places the computer looks to find programs. To verify this, run

pandoc --version

That command should fail. Now run

/usr/lib/rstudio-server/bin/pandoc/pandoc --version

That should tell you stuff about pandoc, demonstrating that the computer and pandoc are communicating.

To convert a markdown file into Word version, you would run:

/usr/lib/rstudio-server/bin/pandoc/pandoc Cats.md -s -o Cats.docx

To get an HTML or PDF version, you would run:

/usr/lib/rstudio-server/bin/pandoc/pandoc Cats.md -s -o Cats.html

/usr/lib/rstudio-server/bin/pandoc/pandoc Cats.md -s -o Cats.pdf

Try those commands on your markdown file. Verify that the outputs are created in the Files pane. Download the generated files via Modify -> Export...in the Files pane. Do they look as they should?
